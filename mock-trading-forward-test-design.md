# 추세추종 4주 KRX 모의 전진검증 봇 + 매매 현황 탭 — 설계

> 상태: **설계(구현 전)**. 백테스트(생존편향·단조성≈0로 단정 불가)의 후속 — 라이브 전진검증.
> 글로벌/프로젝트 규칙 준수: Decimal 전면, 한글 문자열은 `gen-i18n.mjs` 단일출처, swing-bot 파일
> 무수정(패턴만 참고), 시크릿 .env, Conventional Commits.

## 1. 목적 · 범위

- 백테스트한 **추세추종 전략을 KIS 모의계좌로 4주 라이브 전진검증**한다.
- **순수 로직이 모든 의사결정**(추세점수·종목선정·진입/청산·사이징). **KIS 모의 API = 시세 조회 +
  주문 실행**에만. **LLM·외부 판단 0.**
- 봇은 **24/7 상주**, 주문은 **정규장(09:00~15:30 KST)** 에만. 장중 **1분 주기(가능한 한 짧게)** 통합 루프.
- 산출: 실현수익률을 **KOSPI/KOSDAQ 동일기간**과 비교(전진검증 리포트) + **대시보드 매매 현황 탭** 상시 노출.

## 2. 확정된 결정 (브레인스토밍)

| 항목 | 결정 |
|---|---|
| API 역할 | 로직이 전부 결정, KIS 모의 API = **시세 + 주문 실행**만 |
| 매매 규칙 | **백테스트 전략 재현** — 점수 상위 N 동일가중 보유, 순위 밖 교체, **매도요구**(트레일링 손절·200일선 이탈) 청산 |
| 빈도 | 장중 **1분 주기 통합 루프**(레이트리밋·일봉전략 한도 내 최대한 짧게) |

## 3. 확정 (사용자 지시 2026-06-19)

- **계좌**: **swing-bot 모의계좌 50190719 재사용**(시드 **5억**). → **선결: swing-bot 중지**(사용자
  운영 작업 — 나는 그 프로덕션 봇을 건드리지 않음) + **기존 보유 전량 청산·현금 리셋** 후 인계.
  (두 봇이 한 계좌 동시 운용 시 잔고·주문 충돌.)
- **사이징**: **5억 — 국장 2.5억 / 미장 2.5억(5:5, 운영 중 조정 가능)** · 각 시장 top-N=20 동일가중
  (시장별 1/20 ≈ 5%) · 현금버퍼 ~5%. N은 백테스트와 동일(직접 비교).
- **주문 방식**: **시장가(실전형)** — "실제 시장에서 거래하는 형태". 체결 즉시성↑·슬리피지 현실 반영.
- **유니버스**: **국장 KR 292(보통주) + 미장 US 상위 N**(yfinance 429 한도로 '전 종목'은 비현실 →
  `_US_LIQUID` 상위, 기본 400). 두 시장 동시 운용(KR=원화·KIS, US=달러·yfinance/주문경로 별도).
- **타이밍**: **미장 = 구현 완료 즉시 / 국장 = 다음 월요일 09:00**.
- **운영**: OCI 공유, 신규 `trend-trader.service`(대시보드 `trend-board`와 분리).
- **기간**: 연속 4주(영업일 ~20일). 시작 시 종료/리포트 자동.

> ⚠ **선결조건(중요)**: 본 매매봇은 **아직 코드가 없다(설계 단계)**. 대시보드는 시세 '읽기' 전용 —
> **주문 실행부 미구현**. 따라서 "지금 바로 거래"는 불가하며, **P1~P7 구현 + 모의검증 완료 후** 가동한다.
> 실거래(실계좌)는 본 범위 밖(모의 전진검증 전용).
>
> ✅ **미장 주문 = KIS 해외주식 모의 실주문 구축**(사용자 결정). 국내(KR) 연동과 별개로 해외주식 API
> (해외 TR·USD·환전·거래시차)를 신규 구현(§6.2). 두 시장 세션(KR 09:00~15:30 / US 22:30~05:00 KST)을
> 합치면 봇이 사실상 **24시간** 가동된다. 둘 다 **모의(가상돈)** — 실거래 아님.

                  │   1분 루프 ─► 시세갱신(보유∪후보) ─► 손절/로테이션 판정 ─► OrderExecutor ─┐ │
                  │                                                          (신규 KIS 주문)  │ │
                  │   PositionManager(보유·현금·평단) ◄── 체결조회 ◄─────────────────────────┘ │
                  │                  │                                                        │
                  │                  ▼  TradeStore(SQLite, WAL)  orders·fills·positions·nav   │
                  └──────────────────────────────────────┬───────────────────────────────────┘
                                                          │ (읽기 전용)
                        trend-board.service ── /api/trading/* ──► 대시보드 "매매 현황" 탭

- **재사용(무수정)**: `scoring.py`(점수·등급), `stops.py`(트레일링 손절·매도요구), `factors.py`
  (build_candidate), `market_data.LiveProvider`(KIS 시세·토큰 싱글턴·시세 글리치 가드), `config.py`.
- **신규**: `KisOrderClient`(모의 주문/체결조회), `PositionManager`, `StrategyEngine`(목표 포트폴리오
  산출), `TraderLoop`(1분), `TradeStore`(SQLite), `/api/trading/*` 라우트, 프론트 탭.
- **분리 원칙**: 트레이딩 봇이 **쓰고**, 대시보드 API는 TradeStore를 **읽기만**. SQLite **WAL**로 동시
  읽기/쓰기. (대시보드에서 주문 트리거 불가 = 안전.)

## 5. 매매 로직 상세 (백테스트 동치)

- **목표 포트폴리오**: 점수 상위 N(등급 매수·적극매수, 하드필터 통과) 동일가중. 30분마다 풀 재스코어로
  갱신(일봉 팩터는 종가에만 변함 → 분단위 풀 재계산 불필요).
- **1분 루프(장중)**:
  1. **시세 갱신** — 보유 종목 ∪ 현재 top-N 후보 ∪ 경계 워치리스트(top-N×1.5)만(수십 종목 → 레이트리밋
     여유). KIS 시세.
  2. **청산 판정** — 보유 종목에 `stops.py` 적용: **트레일링 손절·200일선 이탈(매도요구)** 발동 시 즉시
     매도. + **순위 이탈**(히스테리시스: top-N×1.5 밖으로 밀리면) 교체 매도.
  3. **진입 판정** — top-N 중 미보유 종목을 빈 슬롯·현금 한도 내에서 매수(목표비중 1/N).
  4. **주문 실행** — OrderExecutor가 지정가 주문 접수 → 체결조회. 미체결은 다음 루프에서 재가격/취소.
- **사이징**: 목표금액 = (총평가액×(1−현금버퍼))/N. 수량 = 목표금액 // 현재가, **호가단위·최소주문금액**
  보정. 초과/부족분은 현금버퍼가 흡수.
- **멱등성**: 클라이언트 주문ID + 미체결 추적으로 **중복주문 차단**. 루프 재진입/재시작에도 안전.
- **부분체결**: 잔량 추적, 평단 재계산.

## 6. KIS 모의 주문 연동 (신규 — swing-bot 패턴 참고, 독립 구현)

### 6.1 국장(KR) — KIS 국내 모의 실주문

- **도메인**: 모의(`KIS_MODE=mock`). 토큰은 LiveProvider 싱글턴+디스크 영속 패턴 재사용(별도 토큰 파일).
- **TR**: 현금 매수/매도(모의 TR_ID), 정정/취소, 잔고조회, 주문체결조회. (정확한 TR_ID는 구현 시 KIS
  문서 확인 — 모의/실전 상이.)
- **체결 신뢰성 가드**(메모리 교훈): **모의 장외 "filled"=유령 체결**(정산 안 됨) → **체결 인정은
  장중만**. 잔고조회로 실보유 재확인(reconcile).
- **시세 글리치 가드**: LiveProvider 기존 ±50% 재조회 로직 재사용.

### 6.2 미장(US) — KIS 해외주식 모의 실주문 (신규 — 큰 별도 작업)

- **주문/체결/잔고 TR**: 해외주식 전용 TR(국내와 상이). 거래소 코드(NASD/NYSE/AMEX), 통화 **USD**.
  미국 정규장 + (지원 시)주간거래.
- **거래시간**: 미 정규장 **22:30~05:00 KST**(서머타임 ±1h) → KR 세션과 합쳐 봇 24시간 가동.
- **환전/증거금**: 모의 USD 예수금/통합증거금(원화↔USD) 정책 확인 + **시드 5억의 KR/US 배분** 정책.
- **시세**: 라이브는 KIS 해외 시세 TR(지연 가능). 백테스트는 yfinance(별개).
- **벤치마크**: S&P500(^GSPC). (KR은 KOSPI/KOSDAQ.)
- **주의**: 가장 무거운 구현 항목 — 약정·환전·시차·TR 차이. "지금 바로" 불가, 별도 Phase.

## 7. 리스크 · 에러 처리

- **장 상태**: 개장 전/마감 후/휴장 주문 차단(`market_hours.py` 재사용). 동시호가(08:30~09:00,
  15:20~15:30) 정책 명시(진입은 09:00 이후, 손절은 정규 체결 시간 내).
- **주문 실패/거부**: 사유 로깅 + 백오프 재시도. 연속 실패 시 해당 종목 일시 제외(전체 중단 아님).
- **graceful degrade**: 일부 종목 시세 실패 → 그 종목만 skip(루프 지속). KIS 토큰/네트워크 장애 →
  주문 보류, 보유는 유지, 다음 루프 복구.
- **킬 스위치**: `.env` 또는 파일 플래그로 신규 진입 중단(보유 청산만 허용) / 전체 정지.
- **계좌 정합성**: 매 루프 시작 시 KIS 잔고로 PositionManager 보정(메모리-실계좌 드리프트 차단).

## 8. 측정 · 검증

- **TradeStore(SQLite, WAL)** 테이블:
  - `orders`(주문ID·종목·방향·수량·가격·상태·시각·사유)
  - `fills`(체결 단위·수량·가격·수수료)
  - `positions`(스냅샷: 종목·수량·평단·시각)
  - `nav_daily`(일별: 총평가액·현금·실현손익·누적)
- **지표**: 일별 NAV → 총수익·CAGR환산·**vs KOSPI/KOSDAQ(동일기간, yfinance ^KS11/^KQ11)**·MDD·승률·
  회전율·평균보유일. 4주 종료 시 리포트(MD/JSON) 자동 생성.
- **백테스트 대비**: 동일 규칙이므로 라이브 결과를 백테스트와 병치(전진검증의 핵심).

## 9. 대시보드 "매매 현황" 탭 — 설계

- **탭 추가**: `MarketTabs`에 `tab.trading`("매매 현황") 추가. i18n은 `gen-i18n.mjs` 단일출처(한글
  직접삽입 금지). 면책 문구 상시 노출.
- **구성(위→아래)**:
  1. **요약 헤더** — 계좌 총평가액·현금·총손익(실현+평가)·일손익·가동상태(LIVE/정지)·4주 진행률(D+n/20).
  2. **NAV 곡선** — 포트폴리오 vs KOSPI vs KOSDAQ(누적수익률 라인). 전진검증 한눈 비교.
  3. **보유 포지션 테이블** — 종목명·수량·평단·현재가·평가손익(₩/%)·손절가·보유일·등급/점수.
  4. **최근 주문·체결 타임라인** — 시각·종목·매수/매도·수량·가격·상태·**사유**(진입=점수상위 / 청산=
     트레일링 손절·200일선·순위이탈).
  5. **오늘의 액션 요약** — 금일 진입/청산 건수·회전율.
- **API(신규, 읽기전용)**: `GET /api/trading/status`(요약), `/positions`, `/history?limit=`,
  `/nav`(시계열). 전부 `dynamic='force-dynamic', revalidate=0`(폴링 stale 방지, 메모리 교훈).
  TradeStore를 **읽기만**. 면책 meta 포함.
- **폴링**: 장중 30초~1분, 장외 느리게. 기존 `usePolling` 훅 재사용.
- **안전**: 대시보드는 **표시 전용** — 주문/정지 등 제어는 노출하지 않음(별도 운영 경로).

## 10. 배포 · 운영

- **신규 서비스**: `trend-trader.service`(데몬, uvicorn 아님). 대시보드와 분리.
- **`.env` 추가**(시크릿·gitignore): 모의 appkey/secret, 계좌번호, `TRADER_TOP_N`, `TRADER_LOOP_SEC`,
  `TRADER_CASH_BUFFER`, `TRADER_KILL_SWITCH`.
- **공유 SQLite**: `data/trading.db`(WAL). 대시보드 서비스가 읽기 접근.
- **모니터링**: systemd 상태 + 텔레그램 알림(매도/이상시만 — swing-bot 패턴, 선택).

## 11. 구현 단계 (Phase) — *다음 단계, 본 문서 승인 후*

1. **P1** `KisOrderClient` **국내** 모의 주문/체결/잔고 + 토큰 — 단위테스트(mock)
2. **P2** `PositionManager` + `TradeStore`(SQLite/WAL)
3. **P3** `StrategyEngine`(scoring 재사용) + `TraderLoop`(1분, 티어드)
4. **P4** 리스크/에러/멱등/킬스위치/reconcile
5. **P5** `/api/trading/*` + **P6** 프론트 "매매 현황" 탭(i18n·폴링·NAV차트)
6. **P7** 국장 모의 실배포 → **월요일 가동**
7. **P8(해외)** KIS **해외주식** 모의 주문/환전/거래시간 → 미장 가동 (가장 무거움)
8. **P9** 4주 운영 → 전진검증 리포트(KR+US vs KOSPI/KOSDAQ/S&P500)

각 Phase DoD: `uv run pytest`·`ruff`·`mypy` / 프론트 `npm run build`·`vitest`.

## 13. 확인 요청 (남은 항목)

대부분 확정(§3·§6.2). 남은 것:
1. ✅ 계좌 50190719·5억 / ✅ 시장가 / ✅ KR292+US400 / ✅ US=KIS 해외 — **확정**.
2. **실거래 아님 재확인**: 50190719 = 모의(가상돈) 맞는지. (실계좌면 즉시 중단.)
