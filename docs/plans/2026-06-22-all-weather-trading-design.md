# 올웨더(3레짐) 매매 로직 + 전 종목 1분 시세 — 설계

> 2026-06-22 브레인스토밍 확정. trend-trader(KIS 모의 전진검증)를 약세장에 현금만 대기하지
> 않는 **장세 적응형(all-weather)** 봇으로 확장하고, 국장/미장 탭을 **전 종목 1분 갱신**으로 전환.

## 1. 배경 / 문제

- 현재 트레이더는 추세추종(맛동산/Sperandeo) 단독 → **약세장엔 진입 후보 0 = 현금 대기**가 설계상
  정상이나, 사용자 요구는 "변동성장에서도 능동적으로 수익을 노려라".
- 점수 대시보드(국장/미장 탭)는 30분 갱신 → **1분 갱신** 요구.
- (선결) 라이브 재검증에서 드러난 표시 버그는 별도 처리: 유령 포지션(flat 고정)·주문 종목명·
  체결 재조회 미가동. → 본 설계의 **단계 0**에서 같이 마무리.

## 2. 목표 (사용자 확정: 1+3)

장세별 3레짐 **올웨더**:
- **상승추세** → 추세추종(기존 코어 유지)
- **횡보·고변동** → 평균회귀/변동성 슬리브 (신규)
- **하락** → **인버스 ETF 롱** (신규; KR 개별 공매도 불가 → 인버스 ETF 매수로 구현)

## 3. 아키텍처

```
[전 종목 1분 가격 레이어]  KR 300 + US 30 현재가 1분 (일봉 점수는 일1회 캐시)
        ▼
[레짐 엔진]  지수(KOSPI/S&P) 추세×변동성 → UP_TREND / CHOP_VOL / DOWN  (하이브리드: 지수→종목)
        ▼
[슬리브 라우터]  레짐이 주력 슬리브 1개 선택(전환식) → 종목별 신호로 선별
   ├─ UP_TREND → 추세추종 (기존 로직 그대로 호출)
   ├─ CHOP_VOL → 평균회귀 (RSI2 과매도 반등)
   └─ DOWN     → 인버스 ETF (KODEX 인버스 롱)
        ▼
[리스크 가드]  포지션수·사이즈·일손실 킬스위치·최소보유·손절·전환 쿨다운
        ▼
[주문 실행]  기존 TraderLoop(KIS 모의) 재사용 — 슬리브는 "결정 공급"만
```

**원칙:**
- 기존 추세추종 코어 **무수정**(UP_TREND에서 그대로 호출).
- 슬리브 = 매수/매도 후보 리스트 생성기. 실행·리스크 가드·체결 재조회는 기존 `TraderLoop` 재사용.
- 레짐당 **주력 슬리브 1개 전환식**. 전환 시 반대 포지션 선청산.
- 신규 슬리브는 **백테스트 게이트** 통과 후 라이브(플래그 OFF 기본).

## 4. 레짐 판정 (하이브리드)

**① 지수 레벨** (KR=^KS11, US=^GSPC; 일봉 기준 일1회 산정):

| 축 | 지표 | 판정 |
|---|---|---|
| 방향 | 지수 vs **MA200** | 위=상승편향 / 아래=하락편향 |
| 강도 | **ADX(14)** | ≥25 추세 / <20 횡보 |

- **UP_TREND**: MA200 위 && ADX≥25
- **DOWN**: MA200 아래 && ADX≥25
- **CHOP_VOL**: ADX<20(횡보) 또는 실현변동성 상위(60일 80%ile↑)인데 추세 약함
- 경계(MA200 근처 + ADX 20~25) = **직전 레짐 유지(히스테리시스)**

**② 종목 레벨**: 선택 슬리브 안에서 종목별 신호로 선별(아래 §5).

**③ 휩쏘 방지(핵심):** **레짐은 일봉 마감 기준 일1회 확정**. 장중엔 가격만 갱신(레짐 고정).
장중 급락(예: 지수 −3%)은 "긴급 디리스크"(보유 축소)로만, 신규 인버스 진입은 익일. → 1분 시세는
*트리거·청산*엔 쓰되 *레짐 전환*엔 안 씀.

## 5. 슬리브별 진입·청산

| 슬리브 | 진입 | 청산 | 성격 |
|---|---|---|---|
| 추세추종(UP) | 기존 점수 상위(하드필터 동일) | 무상태 트레일링(60일 고점×0.92, 200일선 위) | 추세를 길게 |
| 평균회귀(CHOP) | **RSI(2)<10 AND 200일선 위**(거래대금 필터 공유) | **MA20 복귀 or RSI>50 익절 / 시간손절 3일 / 하드손절 −5%** | 반등만 짧게 |
| 인버스(DOWN) | DOWN 확정 → KODEX 인버스(114800,1X) 매수(US=SH) | DOWN 이탈(200일선 회복/ADX 약화) → 전량 + 트레일링 | 추세 확인 시만 |

- 평균회귀는 **추세와 정반대로 평균(MA20)에서 즉시 익절**, 3일 시간손절로 칼청산.
- MA200 위 조건 = "떨어지는 칼날" 매수 차단.
- 인버스 **1X 기본**(2X는 변동성 드래그로 횡보장에 녹음 → 확신 DOWN+짧은 보유에만, 사이즈로 통제).
- 레짐 배타적이라 추세 롱과 인버스 동시 보유 없음(전환 시 선청산).
- 평균회귀 신호는 RSI2 기본, **볼린저 하단 복귀**와 백테스트 비교 후 택1.

## 6. 자본배분 · 리스크 가드 (균형 프로파일)

**배분** (레짐당 주력 1슬리브):

| 항목 | 값 |
|---|---|
| 현금 버퍼 | 상시 5% |
| 최대 동시 보유 | 추세/평균회귀 5종목, 인버스 1~2 |
| 종목당 비중 | ≤20% |
| 인버스 총 익스포저 | ≤50%(1X) / ≤25%(2X) |

**가드:**

| 가드 | 규칙 |
|---|---|
| 일손실 킬스위치 | 당일 NAV −3% → 신규 진입 중단(청산 계속), 익일 해제(기존 인프라) |
| 종목당 손절 | 추세=트레일링 / 평균회귀=−5% / 인버스=−7% 트레일링 |
| 과매매 억제 | 같은 종목 **당일 재매수 금지**(멱등 가드 확장) + 평균회귀 최소 보유 1일 |
| 레짐 전환 쿨다운 | 전환 후 **최소 2거래일 유지** |

> 임계값(MA200/ADX25/변동성80%ile/손절%)은 전부 **백테스트 튜닝 대상**.

## 7. 전 종목 1분 가격 레이어

| Tier | 주기 | 내용 |
|---|---|---|
| 무거운 스캔 | 일 1회(캐시) | 유니버스 + 일봉(280봉 페이지네이션) + 일봉팩터 + 레짐 확정 |
| **현재가** | **1분, 전 종목** | KR 300 + US 30 현재가 → 가격민감 항목 재계산(near52w·현재가·트레일링·평균회귀 RSI 트리거) |

- 무거운 건 *일봉*(이미 일1회 캐시), **현재가는 1콜/종목으로 가벼워** 빈도만 30분→1분.
- **KR**: `inquire-price` 300콜을 스레드풀 + TPS 페이싱(실전키 ~20TPS 내 ≤15/초)로 <30s 완료.
  토큰 싱글턴·세션 재사용.
- **US**: 30종목 1요청 배치(yfinance + curl_cffi 임퍼소네이션).
- **백오프·graceful degradation**: 429/TPS/타임아웃 → 직전 가격 유지 + stale 표시 + 다음 분 재시도
  (크래시 금지). 스로틀 시 보유·상위는 보장, 꼬리 종목만 지연되고 **로그/대시보드에 명시**(조용한
  누락 금지).
- 프론트: 폴링 60초, 카운트다운 60초, "마지막 갱신" 1분.
- caveat: KIS 일일 호출량 ~30배, 1-OCPU 부하↑ → 한도/부하 시 degradation으로 꼬리부터 자동 지연.

## 8. 검증 · 백테스트 · 단계 배포

**백테스트 게이트(라이브 전 필수):** `backtest_10y.py` 확장 — 레짐 전환+3슬리브 시뮬(평균회귀 RSI2,
인버스=지수 −1배 프록시). 비교군 ①추세단독 ②KOSPI 바이앤홀드. 지표 CAGR·MDD·Sharpe·승률 +
레짐별 기여도. 한계: 생존편향+모의 체결 → 최종 증명은 4주 전진검증.

**단계 배포(플래그 OFF 기본):**

| 단계 | 내용 | 게이트 |
|---|---|---|
| 0 (지금) | 트레이더 재시작(체결재조회) + 기존 주문 종목명 backfill → 표시 버그 해소 | 즉시 |
| 1 | 레짐 엔진(시황 탭 표시·읽기전용) + 전 종목 1분 가격 레이어 | 거래 무변경, 며칠 관찰 |
| 2 | 평균회귀 슬리브(플래그) | 백테스트 통과 |
| 3 | 인버스 슬리브(플래그) | 백테스트 통과 |

env 플래그: `SLEEVE_TREND`(on)·`SLEEVE_MEANREV`·`SLEEVE_INVERSE`(off)·`REGIME_ENGINE`.

**테스트(DoD):** 레짐 엔진·슬리브 신호·리스크 가드·1분 레이어(페이싱/백오프/degradation) 단위+통합 +
ruff/mypy/pytest + 프론트 빌드/vitest.

**전진검증 연속성:** 올웨더 전환 시점을 NAV 차트에 마킹(추세단독 구간 보존), 수익률 기준선을
**시드(5억)**로 교정(−0.02% 혼란 해소), 추세단독 vs 올웨더 비교.

**배포:** 단계별 새 브랜치→PR→main→서버 `git pull`+restart(board·trader).

**발견(설계 수정 필요)**: RSI2+200일선은 **횡보장 전략이 아니라 "상승추세 눌림목 매수"(Connors식)**다.
CHOP_VOL 에서 음의 기대값, 상승장 눌림에서만 양. 게다가 강세장 buy&hold(+239%)엔 크게 미달(반등만
먹고 추세를 놓침). **약세장 무활동 문제는 평균회귀로 못 푼다(200일선 위 조건).**

**결론**: 설계대로 "CHOP_VOL→평균회귀" 라이브 활성화 **부적절**(게이트 미통과). 슬리브 코드+백테스트는
인프라로 보존(플래그 OFF, 라이브 미배선). 다음 중 택1로 설계 수정 후 재게이트:
- (a) RSI2 를 **UP_TREND 눌림목 어더**로 재배치(추세추종과 보완, 유휴현금 활용) — 단 buy&hold 미달 유의.
- (b) 횡보장용 신호를 **볼린저 하단 복귀** 등으로 교체해 재백테스트.
- (c) 약세장 활동은 평균회귀가 아니라 **단계 3 인버스 슬리브**가 담당(권장 우선순위).

## 9. 선결 전제 / 미해결

- **트레이더 재시작** 필요(현재 board만 재배포, 체결 재조회 미가동 → 주문 접수/체결 불일치 잔존).
- 기존 주문 2건 종목명 backfill(코드→이름).
- KIS 모의의 인버스 ETF 매매 가능 여부 1회 실호출 확인(ETF=주식처럼 거래 추정).
- KIS 일별체결조회(VTTC8001R)가 모의에서 체결을 반환 안 하면 체결을 **잔고 델타로 역산** 대안.
- 임계값·신호(RSI2 vs 볼린저)·인버스 1X/2X는 백테스트로 최종 확정.
