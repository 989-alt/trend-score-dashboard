# 매매 로직 검증 방법 (Validation Method) + 진입편향 수정 — 스펙

> 작성 2026-06-17. 목적: "바꾼 로직이 **정확하고(correct) 유의미한(meaningful)** 개선인지"를
> 자기기만 없이 판정하는 검증 틀을 먼저 세우고, 그 위에서만 점수 로직을 바꾼다.
> 전제: §3.5(`HANDOFF.md`) — **수익률 목표로 점수를 튜닝하지 않는다(과최적화 금지).**
> 채택 기준은 CAGR 숫자가 아니라 **MAE↓·단조성↑ + 통계적 유의 + OOS**.

---

## 0. 왜 검증 방법이 먼저인가 (현 측정틀의 공백)

베이스라인(43종목·2020–2024): 단조성 ≈ 0, MAE −4%→−14%. "현 점수는 forward-return을
예측 못 함"이라는 **진단**으로는 충분하다. 그러나 "바꾼 로직이 진짜 개선인지" 판정하려면
지금 틀에 세 가지가 빠져 있다.

1. **OOS 없음** — v1은 in-sample 전용. 가중치를 만지면 곧 곡선맞춤.
2. **유의성 검정 없음** — 단조성 ±0.02는 거의 노이즈. 풀링 Spearman N≈520에서
   순위상관 표준오차 ≈ 1/√(N−1) ≈ 0.044, 95% 반폭 ≈ 0.086. 즉 베이스라인 단조성의
   신뢰구간은 **0을 포함** → "약한 음의 실력"이 아니라 **측정 가능한 실력이 0**. 어떤
   로직이 *유의하게 양수*가 되려면 단조성이 **대략 +0.1 수준**까지 올라야 하고,
   forward 윈도우가 겹치므로(주/월 리밸런스 × 20·60일 호라이즌) 실효 표본이 줄어 그
   문턱은 더 높아진다 → **닫힌식이 아니라 (날짜) 블록 부트스트랩**으로 검정해야 한다.
3. **테스트베드 약함** — 43종목·테마편중·생존편향. 포트폴리오 +50%는 점수 실력이
   아니다(단조성 0이 증거).

## 1. 검증 방법 — "정확한가 + 유의미한가" 4계층

| 계층 | 질문 | 방법 | 통과 기준 |
|---|---|---|---|
| **A. 계측기 정확성**("로직이 정확한가") | 하니스가 *실력*을 충실히 측정하나? | **합성 그라운드트루스 테스트**: ① 설계상 forward-return을 예측하는 합성 신호 → 단조성 강한 양수 회복 ② 순수 난수(시드 고정) → 단조성 ≈ 0(CI가 0 포함) ③ **룩어헤드 카나리아**: 미래(>T) 데이터만 바꿔도 T 시점 점수·랭킹 **불변**이어야 함 | 셋 다 기대대로 → 이후 숫자 신뢰 |
| **B. 통계적 유의성**("유의미한가") | 관측 개선이 노이즈인가? | **날짜 블록 부트스트랩** 95% CI(리밸런스일 단위 리샘플 → 풀링 Spearman·평균 MAE 재계산) + **퍼뮤테이션 귀무분포**(날짜내 점수 셔플 → 무실력 단조성 분포) + **페어드** MAE 차이 CI(동일 이벤트, 두 점수셋) | 개선의 95% CI가 **베이스라인 배제**(0 배제만으론 부족) |
| **C. 표본외 일반화**("곡선맞춤 아닌가") | 미래에서도 버티나? | **앵커드 워크포워드**(확장 윈도우 train/test). 파라미터·프리셋 선택은 train 폴드에서만, 채택 지표는 **untouched test 폴드**에서만. 최근 12–18개월 **홀드아웃**은 마지막 1회만 | 채택 지표는 **OOS에서만** 산정 |
| **D. 정직성 가드** | 편향이 결론 오염? | 동일 유니버스 **A/B**(상대비교는 공정) + 시점별 상장종목 유니버스 확장은 병렬 데이터 트랙. 생존편향·테마편중 caveat 리포트 명시 | 절대수익 아닌 **상대 A/B + OOS**로만 결론 |

**다중검정 규율:** 가설 사전등록(near_52w↓ + pullback_3pos + extension_guard ⇒ 20·60일
MAE 축소). 시험 변종 수 제한, 홀드아웃은 마지막에 1회만 연다.

### 채택 규칙 (모든 로직 변경의 게이트 — 사용자 확정: "엄격")
> 로직 변경은 **OOS 테스트 폴드 + 동일유니버스 A/B**에서 ① 평균 MAE 축소 **그리고**
> ② 단조성 회복 — **둘 다** 개선, 그리고 ③ 그 개선의 **블록 부트스트랩 95% CI가
> 베이스라인을 배제**(통계적 유의)할 때**만** 채택. 아니면 **롤백**. **CAGR는 채택 기준
> 아님.** 많은 변경이 기각될 수 있고, 그것이 정직한 결과다.

## 2. 태스크 분담 (순차 — 각 태스크 구현→스펙리뷰→코드품질리뷰)

검증 인프라(T1–T3)가 **로직 변경(T5)보다 먼저**다. 임계경로(T1·T2·T3·T5 단위테스트)는
합성 픽스처로 **비밀키·네트워크 없이** 완결. DART/네트워크는 실데이터 재실행(T4·T6,
T5 최종 채택평가)에만 필요.

| # | 태스크 | 계층/출처 | 비밀키·네트워크 |
|---|---|---|---|
| **T1** | 하니스 자가검증 테스트(신호회복·노이즈0·룩어헤드 카나리아) | A | 불필요(합성) |
| **T2** | 유의성 모듈(날짜 블록 부트스트랩 CI + 퍼뮤테이션 + 페어드 MAE) → 리포트 배선 | B | 불필요 |
| **T3** | OOS 앵커드 워크포워드(train/test + 홀드아웃) — `run.py`/리포트 | C | 불필요 |
| **T4** | `quality_tilt` 비교 실행을 **새 게이트로** 판정 | 게이트 첫 적용 | 필요 |
| **T5** | 진입편향 수정: `pullback_3pos`+`extension_guard`+`near_52w .30→.18` → 게이트 통과 시만 채택 | 로직 | 단위테스트 불필요/채택평가 필요 |
| **T6** | 데이터 보강: value PER/PBR(KRX 인증) + rev_growth DART 전년동기 파싱 | D | 필요 |

### T1 상세 (이번 세션 시작점)
신규 `tests/test_backtest_validation.py`(+ 필요시 `tests/fixtures/backtest_synth.py` 헬퍼).
**프로덕션 코드(`backend/`) 무변경**(순수 테스트). 다음 5개 테스트:
1. **planted positive** — `spearman_monotonicity`에 단조 증가 (점수,fwd) 입력 → 강한 양수(≈1),
   역전 입력 → 강한 음수. 메트릭 부호·스케일 검증.
2. **noise → 0** — 시드 고정 RNG로 독립한 점수·fwd(n≥400) → `abs(단조성)`이 작음(노이즈).
3. **end-to-end 신호 회복** — K개(예 12) 합성 종목: 잠재강도 sᵢ가 (a) 트레일링 구간 기울기(→
   near_52w/momentum 점수)와 (b) forward 수익률을 **둘 다** 결정하도록 가격경로 구성(하드필터
   통과: 거래대금·변동성밴드·200일선·모멘텀≥0). `run_backtest` → `event_study[h].monotonicity`
   뚜렷한 양수 & n>0. **하니스가 심은 신호를 회복**함을 증명.
4. **end-to-end 노이즈** — 트레일링 강도와 forward 방향을 분리(시드 RNG) → 단조성 0 근방.
   **하니스가 신호를 조작하지 않음** 증명.
5. **룩어헤드 카나리아** — 동일 패널에서 T 시점 `_score_at(panel,T)` 계산 → 미래(>T) 행만
   크게 변형한 패널의 `_score_at(tampered,T)`와 **동일**해야 함(점수·랭킹 불변). 더불어 forward
   수익률은 바뀌었음을 확인(변형이 실제 적용됐다는 sanity). **룩어헤드 0** 형식화.

DoD: `uv run pytest`(231 + 신규 모두 green) · `uv run ruff check` · `uv run ruff format` ·
`uv run mypy backend/`(무변경, 통과). 커밋: `test(backtest): 하니스 자가검증 (Layer A)`.
