> 작성 2026-06-16 · 브랜치 `docs/score-reweight-rationale` · 레포 989-alt/trend-score-dashboard
> 방법: subagent-driven 5렌즈 리서치(추세코어·퀄리티·가치·종가매매·리스크/레짐) → 2-lens 리뷰(사실검증 + 교차일관성) → 메인 종합.
> 함께 보는 산출물: `D:\바이브코딩\trend-score-research\{PROGRESS_AND_ROADMAP,STRATEGY_REFERENCE,matdongsan_report}.md`, `quantlab_js/`.

## 0. 이 문서가 정하는 것 / 정하지 않는 것

- **정한다:** 전략·데이터별 매수/매도 *기준*, 각 팩터의 가중치 *prior + 허용범위*, 그 *근거*, 그리고 5개 렌즈를 합친 **기본 점수식의 리웨이트 근거**(합=1.0으로 닫힌 prior).
- **정하지 않는다(=백테스트가 정함):** 가중치의 **최종 단일값**. 본 문서의 숫자는 전부 prior다. 로드맵 §7 게이팅 원칙 — *"백테스트가 베이스라인 대비 개선(특히 MAE 축소·단조성 회복)을 보일 때만 채택, 아니면 롤백."*
- **베이스라인 정정(리뷰 발견):** 현행 점수식은 `config.py` 기준이다. `STRATEGY_REFERENCE.md §0`의 `momentum·0.25`는 **구버전(stale)** — 실제로는 이미 `momentum 0.13 + rs 0.12`로 분할돼 있다(collinearity 회피). 베이스라인은 항상 `config.py`/`PROGRESS §2`를 인용한다.

## 1. 문제 → 리웨이트의 목적

사용자 통증: **"매수 추천 종목을 샀는데 항상 매수 직후 떨어졌다."** 증거기반 진단(로드맵 §2):
1. 점수가 forward-return으로 **검증된 적 없음**(가중치는 swing-bot 기본값 이식, 데이터로 적합한 적 없음).
2. 점수가 눌림목이 아니라 **과열/신고가 극단을 보상** — `near_52w`가 최대가중(0.30)이라 *이미 많이 오른* 종목이 최고점. 맛동산 **3번자리(돌파→눌림→지지→재상승)** 미구현 + 과열 페널티 없음 → **고점 추천 → 매수 직후 눌림**.
3. **시장 레짐 게이트 부재** — 조정장에도 STRONG_BUY 발화.

→ 리웨이트의 목적은 "팩터 더 넣기"가 아니라 **① near_52w 과중 해소 ② 눌림목 보상·과열 차단 ③ 레짐·리스크 통제**. 수익 보장은 불가(사용자 동의) — 검증·리스크통제·고점추격방지가 목표.

### 3.2 near_52w 인하 + pullback 재배분 (리웨이트의 핵심 한 줄)

증상="매수 직후 하락" → 직접원인 = `near_52w`가 최대가중(0.30)으로 **"신고가 도달=과열"을 보상**. 맛동산은 신고가권을 긍정하되 **신고가 *도달 순간*(2번자리)이 아니라 *돌파 후 눌림 지지*(3번자리)** 에서 진입한다(matdongsan §1-나). 따라서:

- `near_52w` 0.30 → **0.18** (신고가 신호 자체는 유지, 과중만 해소)
- 비운 0.12를 **trend_template(추세 질) + pullback_3pos(진입 타이밍)** 로 이전
- 보존 검산(모든 증감은 §2 `config.py` 베이스라인 대비): 인하분(near52w −0.12 · turnover −0.07 · vol_fit −0.10 · momentum −0.03 · rs −0.05 · pp −0.08 = **−0.45**) = 신규(trend_template +0.14 · volume_surge +0.08 · vcp +0.08 · quality +0.08 = +0.38) + **pullback +0.07** → 정확히 닫힘.

**∴ pullback_3pos는 "독립 신규 가산"이 아니라 "near_52w 과중에서 옮겨온 가중"이다.** 이 해석이 산술적으로 합을 닫고, 설계의도(고점보상→눌림보상 이동)와도 일치.

### 3.3 pullback_3pos / extension_guard 결합방식 (부호상쇄 회피)

같은 가산 메커니즘에 두면 extension의 음(−)이 near_52w·momentum의 양(+)과 **상쇄**되어 고점추격 차단에 실패한다. **비대칭 결합:**

| 팩터 | 결합방식 | 적용 위치 | 이유 |
|---|---|---|---|
| **pullback_3pos** | 가산 가점 (a) | 가중합 내부(§3.1 표) | 위로 끌어올림. 기존 min_max+가중합 메커닉 재사용 |
| **extension_guard** | **곱셈 페널티 (c) + 극단 소프트게이트** | **가중합 산출 후 score(0~1)에 곱** → `score_final = Σ(wᵢ·factorᵢ) × ext_factor`, `ext_factor∈[0.5,1.0]`; 극단(예 price>MA20×1.5)은 eligible 영향 | 곱셈은 **상쇄 불가** — 다른 팩터가 아무리 높아도 과열이면 점수가 눌림. "고점추격 차단" 목적에 직접 부합 |

`ext_factor`는 이격도(`(price−MA20)/MA20` 또는 ATR20 배수)가 임계 초과분에 비례해 1.0→0.5로 감쇠. **매수점수에만 적용 — 매도축에 전이 금지**(matdongsan §3 "과열이라 전량매도 말 것"과 충돌).

### 3.4 trend_template: 게이트 vs 0~8 점수 → **0~8 점수(하이브리드) 권고**

- 8of8 하드게이트는 적격종목을 과도하게 좁혀 후보고갈·경계 급단절 위험.
- 권고: **①(P>MA200)은 기존 하드필터로 잔존**, 나머지 7조건을 0~7 점수화하는 하이브리드 → 중복(기존 200일선 필터) 없이 질적 그라데이션. (게이트 vs 점수 최종형은 §8 백테스트 비교.)

### 3.5 collinearity 관리 (반드시)

- **삼중 RS 노출:** `momentum_norm` · `rs_norm` · `trend_template ⑧(rs≥0.70)` 가 모두 같은 모멘텀 신호. → `rs_norm` 0.07로 인하 **또는** trend_template에서 ⑧ 제외(0~7/7). (둘 중 택1 — 백테스트.)
- **거래량 이중:** `pocket_pivot`(이산 0/1) + `volume_surge`(연속) → **합 ≤ 0.20 상한**.
- **변동성 이중:** `vcp_tightness` ↔ `vol_fit` → **vcp 단독, vol_fit=0**(§8 확정).

## 4. 퀄리티·성장 틸트 (메인 점수 0.08)

**역할 분리 = 가산 틸트 + 게이트 둘 다.**
- **가산 틸트(0.08):** ROE·영업이익률·성장(매출·EPS YoY) → 후보군 횡단면 백분위 합성(quantlab식 rank→가중합, 기존 `min_max_norm` 재사용). 근거: Quality+Momentum 양의 보완(퀄리티가 모멘텀 급락 방어, STRAT §3), matdongsan "잡주 제외·실적 주도주만"(L61).
- **v1 권고:** 다요소(F-Score/QMJ) 대신 **단일 robust(Novy-Marx GP/A 또는 ROE) + 성장 1종**. F-Score는 9입력+전년동기라 커버리지·look-ahead 위험 최대 → 실측 후 결정.
- **적자 게이트는 렌즈5 소유**(아래 §7.3) — 본 틸트는 "강한 종목 가산"만.

### 4.1 데이터 배선 — US/KR 비대칭 (리뷰 핵심 정정)

현행 `Fundamentals`(market_data.py): `market_cap·per·pbr·eps·w52_high·w52_low·return_1y_pct·sector·industry·name` **뿐**. ROE·마진·성장 **전부 미배선**.

| | US (yfinance `.info`) | KR (KIS) |
|---|---|---|
| ROE/마진/성장 | `.info` 키 추가만(returnOnEquity·operatingMargins·profitMargins·revenueGrowth·earningsGrowth) → **거의 0배선** | **재무비율 엔드포인트 신규 필요**(로드맵 §5f). 구현 전까지 KR 커버리지 0 |
| 갱신·캐시 | `DailyCache` kind=`fundamentals` **일1회 캐시 존재** | **일1회 펀더 캐시 없음** — inquire-price 30초 TTL 공유. 퀄리티 필드는 일1회 prep 경로 신규 필요 |

⚠ **"퀄리티 배선 0"은 US 한정.** KR은 신규 API + 일캐시 경로 둘 다 필요 — 공수 과소추정 금지.
- **fail-open + 시장별 백분위:** 펀더 결측 종목은 틸트 미적용. KR/US 커버리지 비대칭이 같은 0~100 척도를 왜곡하지 않도록 **백분위를 시장별로 산정**(현 스냅샷이 KR/US 분리라 자연 정합) + 빠진 컬럼 **재정규화**.
- **갱신주기:** 재무는 분기 공시 → **일1회로 충분**(30분 intraday는 캐시 재사용 → US `.info` 429 부담 격리).

## 5. 가치 (별도 렌즈 — 메인 점수 기여 0)

**메인 추세 점수에 섞지 않는다.** 근거 4종:
1. **Value+Momentum 음의 상관** — 같은 가중합에 더하면 신호 상쇄(STRAT §3).
2. **저PER은 구조적으로 신고가 이탈주를 보상** — PER=가격/EPS라 가격↑이 PER↑ → 저PER 상위 = 신고가에서 빠진 소외주 → `near_52w`(0.18)와 정면 충돌.
3. **맛동산 철학 배치** — "소외 저평가주 회피, 신고가 주도주 홀딩"(STRAT §1).
4. **재분석 확정**(로드맵 §3): 펀더는 *싸기*가 아니라 *강하기*. value는 별도 렌즈.

- **별도 '가치 렌즈' 프리셋:** quantlab 저평가 '밸류+퀄리티균형'에서 가치 3팩터 추출·재정규화 → **PER 0.32 / PBR 0.30 / PSR 0.20 / PEG 0.18**(합 1.0) + Graham 배지(불리언). (퀄리티 ROE/마진은 렌즈4 소관 — 본 렌즈 미포함.)
- **최소 출시 단위(신규데이터 0):** `per·pbr` 가용 → **PER rank + PBR rank + Graham 배지**. PSR(US 저비용/KR 신규)·PEG(렌즈4 성장률 공유) 후순위, EV/배당 옵션 제외.
- **'안 비싸게' 가드:** 메인 기여 **0 유지** 권고 — 과열 방어는 가격기반 `extension_guard`(§3.3)에 일임(PER은 EPS 노이즈에 취약). 가드 필요성은 백테스트 미결로만 남김.

## 6. 종가매매 단기 배지 (별도 단기축 — 메인 점수 미통합)

**분리 근거:** 시간지평 1~5일(메인=중장기) + 청산규칙 정면충돌(추세추종 "익절 무목표" vs 종가 "+2~3% 전량"). 손익비도 ≈1.43:1(고승률·저손익비)로 메인 3:1과 다른 레짐.

- **매수 플래그(`eod_swing_candidate`):** `양봉(Close>Open) AND vol/SMA(vol,20)≥2 AND (MA정배열 OR 신고가돌파 OR Close≥MA5) AND 윗꼬리 (H−C)/(H−L)≤0.3`. 합성 = **핵심은 AND 게이트, 추세위치는 OR, 정렬용 약한 점수**(메인 0~100과 합산 금지).
- **청산플랜(신호일 뿐, 자동주문 아님):** 익절 +2~3% / 손절 −1~2.5% 또는 MA5 이탈 / 보유 1~5일. 진입 가정 = **T+1 시가**(look-ahead 0).
- **격리:** 상따(KR 상한가 근접)·세력주(vol>10×=미끼)는 **별도 고위험 티어 라벨**. 동시호가 잔량·틱은 **데이터 부재 → 미구현**.
- **KR/US 분기:** 양봉·거래량·정배열·돌파는 공통; 상한가(±30%)·VI는 KR 개념(단 잔량 미수집이라 상따 격리만 실재).

## 7. 리스크·레짐·잡주 게이트 (매도/필터 축 — 점수를 modulate)

### 7.1 리스크 모듈 (정보용 — 기존 매도 유지)
- `ATR20`(TR=max(H−L,|H−Cprev|,|L−Cprev|) 20봉 평균), `atr_stop=price−2×ATR20`, `suggested_weight=R%/(2·ATR/price)` 상한 10%. **표시 필드로만 추가**(`ScoreEntry`), 매도로직(트레일링 8%·200일선) **불변**.
- **손익비(rr)는 진입 하드필터로 부적합** — 추세추종은 목표가가 없어 rr 분자 미정의(matdongsan §1-다). 정보표시만.
- ATR 트레일링 전환은 무상태(진입가 없음) 원칙과 충돌 + 베이스라인 교란 → **백테스트에서 "8% 평면 vs 2N ATR" 나란히 비교할 때만** 승격.

### 7.2 시장 레짐 게이트 (신규)
- 분산일 = `지수종가 ≤ 전일×0.998 AND 지수거래량 > 전일`(O'Neil distribution day). 최근 **25거래일 ≥5회 → risk_off**.
- 조치: **STRONG_BUY → BUY 상한 + 배너**(통증 "조정장 STRONG_BUY"를 정확히 차단, 다른 축 보존). `Snapshot.market_regime` 신규.
- 데이터: 지수(^KS11/^GSPC)는 RS용으로 이미 조회 — 단 현 `_index_momentum`은 ~28봉·종가만 → **조회 depth ↑ + 거래량 활용** 필요. 지수 결측 시 risk_on fail-open.

### 7.3 잡주 하드필터 (소유 = 렌즈5, fail-open)
- `per≤0 OR eps<0`(적자) 제외. per/eps **현재 가용 → 신규 배선 0**. 펀더 결측 시 미적용(2차 방어선=거래대금·200일선·변동성).
- ⚠ 확장(Altman Z·자본잠식·CFO)은 **미배선** → Phase 2. **KIS 시총 단위(`hts_avls`) 미검증 TODO** — 시총 하한 게이트는 단위 확정(100배 오차 위험)을 **선결조건**으로.

### 7.4 turnover 위치
- 현재 거래대금이 **하드필터이자 점수팩터(0.15)로 이중** 사용 → 점수쪽이 "거래대금 큰=당일 과열주"를 0.15 가산해 통증을 증폭.
- **prior: 점수 0.15 → 0.08 축소**(게이트는 유지). **대안: 순수 게이트화(점수 0)** → 비는 0.08 재배분(§3.1 ²). 택일은 백테스트.

## 10. 무결성 · 함정 (반드시 준수)
