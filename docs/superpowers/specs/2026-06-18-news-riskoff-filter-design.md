# 뉴스 기반 리스크오프 필터 — 설계 (News-driven Risk-Off Filter)

> 작성 2026-06-18 (세션4 후속). 출발: "실시간 뉴스(텔레그램·RSS)를 수집·분석해 시장반응을 예측, 매매에
> 반영" 요청. 사용자 우선순위 = **신뢰성**. 그래서 *예측 틸트*가 아니라 **검증 가능한 비대칭 방어 필터**로
> 좁히고, **fail-fast**로 v1을 "액션 가치 백테스트"에 한정한다.
> ⚠ 원칙(§3.5): 수익률 목표 튜닝 금지. 채택은 *위험조정 OOS 개선*으로만. 효과 없으면 정직하게 보고하고 중단.

---

## 0. TL;DR

- **역할(사용자 확정):** 뉴스를 **리스크오프 필터**로 — 대형 악재 시 *신규매수만* 억제(비대칭 방어, 레짐
  게이트의 확장). "시장반응 예측"(이미-반영·약신호·LLM 환각 함정)은 채택하지 않는다.
- **검증(사용자 확정 — 분리 검증):** **(a)** 리스크오프 *액션*의 가치 = 객관·외생·실시간 트리거 + 큐레이션
  위기일자로 **백테스트(룩어헤드 0)**. **(b)** 뉴스 *탐지기*의 정확도/적시성 = 과거 백테스트 불가 →
  *지금부터 전향(forward)* 측정. v1은 **(a)만**.
- **fail-fast:** (a)가 "악재일 리스크오프가 MDD를 못 줄인다"면 → **뉴스/LLM 파이프라인을 짓지 않는다**(비용 0).

## 1. 신뢰성 프레이밍 (왜 이 형태인가)

"뉴스 읽고 시장반응 예측해 매매"의 3대 신뢰성 킬러:
1. **백테스트 정직 불가** — OOS 검증엔 *시점별(point-in-time) 과거 뉴스 아카이브*가 필요. 오늘 수집하는
   텔레그램/RSS로 과거를 돌리면 룩어헤드. → 그래서 **객관 트리거(시점별 가용)로 액션을 검증**하고, 뉴스
   탐지기는 *전향*으로 따로 검증한다.
2. **이미 반영 + 지연** — 공개 뉴스 *반응* 매매는 유동성 시장에서 엣지≈0. → 그래서 *예측*이 아니라
   *방어(리스크오프)*. 방어는 "초 단위 선반영"과 무관하게 *구간 노출 축소*로 가치를 낸다.
3. **결정론·zero-LLM·₩0 충돌** — LLM은 비결정·환각·비용. → v1은 **LLM 0**(객관 트리거만). LLM은 (b)에서만,
   그것도 *탐지기*로 한정하고 전향 검증.

**핵심 분해:** (a) *액션이 가치 있나*(객관 트리거로 백테스트 가능) vs (b) *탐지기가 정확·적시한가*(전향).
v1은 (a)에 집중 — 아이디어 가치를 먼저, 싸게, 결정적으로 가린다.

## 2. 트리거 — 다채널 객관·외생·실시간 신호 (VIX 단일이 아님)

VIX는 *글로벌/미국발* 리스크만 잡고 **국장 고유·비미국 충격을 놓친다**(환율·지정학·국내정치·반도체·중국·
유가). 그래서 v1 트리거는 *소수의 채널-프록시 바스켓*(각기 다른 위험 채널, **사전 명시**, 표준 임계 소수만):
- **VIX** (`^VIX`): 글로벌 주식 공포.
- **원/달러 급등** (`KRW=X`): 환율 스트레스 = 외국인 이탈(국장 직격).
- **(선택) 미 10년물 급등 / 달러인덱스(DXY)**: 금리·달러 쇼크.
- **(기존·재사용) 코스피 분산일 레짐 게이트**(`regime.py`): 국내 가격/거래량.
- 각 트리거 = 객관 규칙(수준 초과 OR 1일 급변 > X), **≤T 데이터만**(룩어헤드 0). yfinance 캐시 로드.
- **임계는 표준값 소수만**(예: VIX 30/35, 1일 급등 +x%) + 홀드아웃 — 수익률에 맞춰 튜닝 금지(§3.5).

## 3. 큐레이션 위기-이벤트 리스트 = 커버리지 벤치마크 (트리거 아님)

문서화된 글로벌·**한국 고유** 악재 구간(코로나 2020·금리쇼크/우크라 2022·엔캐리 2024-08·**계엄 2024-12**·
북한 도발 등)을 `data/crisis_events.yml`에 둔다. 용도는 *트리거가 아니라 측정자*:
- **객관 트리거가 이 사건들을 몇 % 잡고 몇 %를 놓치는가**(coverage / miss-rate).
- **놓치는 비율(특히 한국-고유) = 나중 (b) 뉴스탐지기가 메울 가치의 정량화.** ← 핵심 산출물.
- 큐레이션은 후견편향 위험 → *객관 편입기준*(예: 당시 주요지 1면급 글로벌/국내 시장 사건)을 명시하고,
  민감도(리스트 가감)로 강건성 확인. 큐레이션은 *검증 대상이 아니라 측정자*임을 명확히.

## 4. 액션 & 검증

- **액션(비대칭):** 리스크오프 일자엔 신규진입 보류(현금). 기존 `simulate_risk_overlay`를 **주입형
  `risk_off_dates: set[date]`** 를 받도록 일반화(기존 분산일 레짐 토글과 **OR 결합**). 손절/사이징과 독립.
  ※ **`risk_off_dates`는 각 리밸런스일 T의 ≤T 외생데이터로 산출(as-of)** → T→T+1 *신규진입만* 억제(동일봉
  룩어헤드 없음 — 기존 레짐 게이트가 `index_rows_asof(t)`를 쓰는 것과 동일 규약). 즉 트리거 산출 함수는
  "리밸런스일 리스트 + ≤각T 외생 시계열 → risk_off 일자 부분집합"이어야 한다.
- **검증(OOS·위험조정):** baseline / +KR레짐 / +VIX / +환율 / 조합 을 **워크포워드 연속 구간**에서
  `portfolio_metrics`(MDD·Calmar·Sharpe) + 레짐 구간 손실로 비교. (경로지표 한계는 fallback-C RESULTS와 동일
  — 상대 비교가 신뢰 신호.)
- **핵심 측정:** *기존 KR 분산일 레짐에 글로벌 트리거(VIX·환율)를 더하면 MDD가 추가로 주는가*(가산성).

## 5. Ablation & 판정 게이트

- **Ablation:** 트리거별 증분 ΔMDD/ΔCalmar/ΔSharpe(fallback-C 레이어2 동형). **효과 없거나 악화 트리거는
  채택 안 함**. 임계 후보 소수 + 홀드아웃 1회.
- **판정 게이트 — (b)를 지을지 결정:**
  1. **액션 가치:** 글로벌 트리거가 KR 레짐 대비 MDD를 *추가로* 유의/명확히 줄이는가?
  2. **커버리지 갭:** 큐레이션 *한국-고유* 사건 중 객관 트리거가 놓치는 비율이 큰가?
  - **둘 다 yes → (b) 뉴스탐지기 구축 정당화**(별도 스펙). **①이 no → 파이프라인 안 짓고 정직 보고**(fail-fast).
- 라이브 승격(통과 시·별도): 객관 리스크오프 트리거를 라이브 엔진/프런트(레짐 배너)에 반영 — 라이브 회귀 필수.

## 6. 테스트 (엔진 정확성)

- **합성 그라운드트루스:** 트리거가 발화하는 *심은 위기구간 + 그 구간 종목 급락* 패널 → 리스크오프(주입
  일자집합 on)가 off 대비 MDD를 줄임을 증명(기존 레짐 그라운드트루스 동형, 비퇴화 — 리스크오프가 *실거래 중*
  크래시만 회피).
- **단위:** 각 트리거 일자 산출(객관 규칙·≤T), `risk_off_dates` 주입이 신규진입을 막는지, 커버리지 지표
  (큐레이션 ∩ 트리거 일자, miss-rate).
- **결정론**(시드/캐시 고정)·**Decimal 전면**·**라이브 무수정 회귀**. DoD: pytest·ruff·ruff format·mypy.

## 8. 범위 · 후속 (3개 하위 프로젝트로 분해)

전체는 3개 하위 프로젝트. **현재 우선순위는 변경됨**(사용자, 2026-06-18): 뉴스 작업은 *계획만 기록*하고
**US 백테스트를 먼저** 진행.

1. **(a) 액션 백테스트 — 본 스펙.** LLM 0·뉴스 0. 객관 트리거(VIX·환율…) + 큐레이션 커버리지로 *액션 가치
   + (b) 빈틈*을 정량화. (fail-fast 게이트.)
2. **수집 파이프라인 — 계획만 기록(deferred, 지금 구현 안 함).** RSS/Telegram → **원시 아카이브**(원문+타임스탬프
   +소스+중복제거, **수집시 LLM 미사용 = ₩0 유지**; 구조화/LLM은 (3) 탐지기에서만). *지금 일찍 짓는 게 정답*인
   이유 = 시점별 과거 뉴스는 살 수 없고 *전향 수집*으로 쌓아야 하니, 빨리 돌릴수록 (3) 전향검증을 빨리 시작.
   **단 매매 무영향**(데이터 인프라 한정) — (a)+전향검증 통과 전까지 매매에 안 닿음. 착수 시 별도 브레인스토밍
   (소스 큐레이션·호스팅(OCI 공유)·저장 스키마).
3. **(b) 탐지기 + 매매통합.** LLM 분석 → 리스크오프 신호 → 매매. (1) 통과 + (2) 아카이브로 전향검증 통과 시.

**현 진행 순서(사용자 확정):** ⟶ **US 주식 동일 백테스트(지금)** ⟶ PIT 시점별 유니버스 + 현실적 비용 보강
⟶ (이후) 뉴스 (2)/(1)/(3). 본 스펙(a)와 뉴스 (2)는 *기록 보존*, 재개 가능.

## 9. 리스크 / 정직성

- **방어는 기회손실 동반** — 상승장에서 리스크오프는 *수익을 깎는다*(MDD↔수익 트레이드오프). 위험조정으로 평가.
- **트리거 임계 과최적화 위험** — 소수 표준값·홀드아웃·민감도로 억제. 트리거 다수 시도 = p-hacking 주의.
- **큐레이션 후견편향** — 객관 편입기준 명시, 검증대상 아닌 측정자로 한정.
- **경로지표 약한 유의성**(fallback-C와 동일) — 상대 비교가 신뢰 신호.
- **수익 보장 없음.** v1은 *액션 가치 가설 검증*이지 수익 약속이 아니다.
