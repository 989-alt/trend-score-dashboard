# 뉴스 리스크오프 #1 — 판정 RESULTS (객관 트리거 액션 백테스트)

> 2026-06-18. 설계: `2026-06-18-news-riskoff-filter-design.md` (#1 = fail-fast 액션 가치).
> 실행: `python -m backend.backtest.run --news-riskoff --start 2019-01-01 --end 2024-12-31
> --rebalance monthly --tickers <유동 KR 37종>`. LLM·뉴스 0, 라이브 스코어러 무수정, 룩어헤드 0.

## TL;DR — **게이트 ① FAIL → 뉴스 탐지기(#3) 미구축이 정답(fail-fast).**

객관 트리거(VIX·환율) 리스크오프는 **기존 분산일 레짐 대비 추가 위험감소가 없다.** VIX 단독은
오히려 해롭다(회복장 잘못 현금화). 한국 고유 충격(계엄)은 객관 트리거가 못 잡는다(0/1). 설계 §5
판정 게이트 ①("글로벌 트리거가 KR 레짐 대비 MDD 를 *추가로* 줄이나?")이 No → **LLM 뉴스 탐지기
파이프라인을 짓지 않는다.** 분산일 레짐이 이미 유효한 리스크오프 장치다.

## 액션 ablation (위험조정 — 2019~2024, monthly, WF 세그먼트 체인)

| config | MDD | Sharpe | Calmar |
|---|---|---|---|
| baseline (오버레이 없음) | −46.0% | 0.42 | 0.19 |
| **+regime (분산일)** | **−27.8%** | 0.34 | 0.18 |
| +vix | −51.7% | 0.07 | −0.03 |
| +fx | −46.0% | 0.42 | 0.19 |
| +regime+vix+fx | −26.9% | 0.34 | 0.18 |

- **분산일 레짐 = 큰 MDD 승**: −46.0% → −27.8% (낙폭 ~40% 축소). (fallback-C 결과 재확인.)
- **VIX 단독 = 해로움**: MDD −51.7%(baseline보다 나쁨), Sharpe 0.42→0.07. 공포 임계로 현금화한
  구간들이 *회복장*이라 수익을 깎고 낙폭도 못 줄였다.
- **환율 단독 = 무효과**: monthly 리밸런스일에 >1.5% 원/달러 급등이 한 번도 안 걸려 트리거 0회
  (= baseline 동일). (일중 스파이크가 월 1회 표본점과 어긋남.)
- **레짐+VIX+환율 ≈ 레짐**: −26.9% vs −27.8% (노이즈 범위). 글로벌 트리거가 레짐에 더하는 게 없다.

## 커버리지 (큐레이션 위기 대비 객관 트리거 VIX∪환율)

- 전체: **2/4** catch. 한국 고유(kr): **0/1** catch.
- 놓침: **엔캐리 청산 블랙먼데이**(VIX 38.57@2024-08-05 — 일자로는 잡히나 *monthly 리밸런스일과
  어긋나* 미반영), **한국 비상계엄 사태**(VIX/환율로는 본질적으로 미포착 = 한국 고유 갭).
- 한국 고유 갭은 존재하나, 게이트 ①(액션 자체의 가치)이 No 라 **갭을 메울 #3 의 정당성도 없다**
  (가치 없는 액션의 커버리지를 늘려봐야 무의미 — 설계 §5 "①이 No → 미구축").

## 한계(정직)

- **monthly cadence**: VIX 스파이크(2024-08-05)가 리밸런스일과 어긋나 미반영. weekly 로 더 잡힐
  수 있으나, **VIX 단독이 해로운 것**(잘못된 시점 현금화)이 더 근본 문제라 cadence 를 바꿔도
  방향이 뒤집힐 가능성은 낮다.
- **소universe(유동 37종)·생존편향**: 광범위 PIT 유니버스(로드맵 NEXT)면 수치가 다듬어지나,
  *상대 신호*(레짐 유효·VIX 해로움)는 명확.
- 경로지표 약한 유의성(fallback-C 와 동일) — 상대 비교가 신뢰 신호.

## 결론 / 다음

   파이프라인 #2 = A 의 `news.db` 가 이미 전향 아카이브로 가동 중이므로, *데이터는 계속 쌓되*
   매매 반영은 안 함.)
- **분산일 레짐**은 유효 → 이미 fallback-C 에 반영됨(별도 승격 경로).
- 한국 고유 리스크오프가 정말 필요하면, 객관·시점별 가용 *한국* 트리거(예: KOSPI 분산일은
  이미 사용 / 추가 후보는 별도 설계)로 다시 게이팅해야지, LLM 뉴스 예측으로 가지 않는다.
